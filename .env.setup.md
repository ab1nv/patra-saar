# Environment Setup Guide

## Quick Start

1. **Copy example files to actual files:**
   ```bash
   cp apps/api/.dev.vars.example apps/api/.dev.vars
   cp apps/web/.env.local.example apps/web/.env.local
   cp packages/ingest/.env.example packages/ingest/.env
   ```

2. **Fill in your credentials** (see below for how to get each)

3. **Run the project:**
   ```bash
   make dev
   ```

---

## Getting Credentials

### 1. Google OAuth (GOOGLE_CLIENT_ID + GOOGLE_CLIENT_SECRET)

1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Create or select a project
3. Enable "Google+ API" (Search for it in the API library)
4. Go to **Credentials** → **Create OAuth 2.0 Client ID**
5. Application type: **Web application**
6. Add Authorized redirect URIs:
   - `http://localhost:8787/auth/callback/google` (local dev)
   - `https://your-api-domain.workers.dev/auth/callback/google` (production)
7. Copy **Client ID** and **Client Secret** into `.dev.vars`

### 2. Groq API Key (GROQ_API_KEY)

1. Go to [console.groq.com](https://console.groq.com)
2. Sign up or log in
3. Create a new API key
4. Copy into `.dev.vars`

### 3. BetterAuth Secret (BETTER_AUTH_SECRET)

Generate a random 32+ character string:
```bash
openssl rand -base64 32
# or
node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"
```
Copy the output into `.dev.vars`

### 4. BetterAuth URL (BETTER_AUTH_URL)

- **Local dev**: `http://localhost:8787`
- **Production**: `https://your-api-domain.workers.dev`

### 5. Cloudflare Credentials (for KB ingestion only)

For `packages/ingest/.env`:

1. Go to [Cloudflare Dashboard](https://dash.cloudflare.com/)
2. Account settings → API Tokens
3. Create token with permissions: Account.Workers KV Storage, Account.D1 Database, etc.
4. Get **Account ID** from the sidebar (Overview tab)
5. Copy **API Token** and **Account ID** into `packages/ingest/.env`

| File | Purpose | Copy from |
|------|---------|-----------|
| `apps/api/.dev.vars` | API secrets | `.dev.vars.example` |
| `apps/web/.env.local` | Frontend config | `.env.local.example` |
| `packages/ingest/.env` | KB ingestion | `.env.example` |

## Verification

After filling in `.dev.vars` and `.env.local`:

```bash
# Start API
cd apps/api && npm run dev

# In another terminal, test API
curl http://localhost:8787/api/categories
# Should return: {"categories":[]}

# In another terminal, start frontend
cd apps/web && npm run dev
# Visit http://localhost:3000
```

If you see the landing page and can click "Sign in with Google", you're good to go!
